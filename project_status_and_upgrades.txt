================================================================================
CURRENT PROJECT INCONSISTENCIES & LIMITATIONS (Addressed in 'changes')
================================================================================

1. Security Vulnerabilities
   - Missing Rate Limiting, Input Validation, and Security Headers across APIs.
   - Exposed Google API Key references in configuration files (`netlify.toml`).
   - Missing robust CSRF protection, tight CORS strategies, and Authorization roles.

2. Architecture & Backend Gaps
   - Lack of automated tests (0% test coverage).
   - Basic health check (`/health`) without deep system metrics (e.g. CPU/Memory usage).
   - Absence of comprehensive server-side logging and analytics.

3. Frontend & Authentication
   - Firebase Auth was implemented but skipped crucial security steps like email verification and structured password recovery.
   - Chat history UI lacked search and filtering capabilities.
   - Code structure in files like `layout.tsx` was flagged as heavily AI-generated, lacking organic developer context or comments.

4. Model & Core Engine
   - The self-correcting engine lacks a robust data-flywheel and automated training pipeline to continuously fine-tune the model based on interactions.
   - Missing standard REST endpoints for users to submit feedback to the reasoning steps.

================================================================================
ROADMAP: NEAR-FUTURE UPGRADES FOR THE FULL PROJECT
================================================================================

1. Implement a Comprehensive Testing Suite
   - Introduce `pytest` for the FastAPI Python backend (unit and integration tests).
   - Use `Jest` and `React Testing Library` for frontend component reliability.
   - Set up End-to-End (E2E) UI testing using Cypress or Playwright.

2. Production Database Integration & Advanced User State
   - Migrate mock historical chats to a robust hosted database (Firestore or PostgreSQL).
   - Persist detailed user preferences and authorization roles securely.

3. Advanced AI Observability & Monitoring
   - Implement specialized LLM observability (e.g. LangSmith or Helicone) to track token usage, generation latency, and path-divergence stats.
   - Hook up a central logger to Datadog or Sentry to quickly catch backend inference crashes.

4. CI/CD Pipeline & Deployment Automation
   - Integrate GitHub Actions to automatically trigger tests, code quality linting, and security scans on every pull request.
   - Containerize both backend and frontend applications using Docker for reliable, infrastructure-agnostic deployments.

5. Dynamic Front-End Feature Enhancements
   - Allow users to explicitly flag individual reasoning steps as "Helpful" or "Inaccurate" to feed the model training pipeline.
   - Add export functionalities (PDF/Markdown) so researchers can easily share generated reasoning chains.
